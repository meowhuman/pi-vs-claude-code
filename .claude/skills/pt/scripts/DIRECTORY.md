## File Organization Rules

### Logs (`logs/`)
- All `.log` files go here
- Automatically ignored by git
- Can be safely deleted to free space

### State (`state/`)
- JSON files tracking persistent state
- **Do not delete** - contains tracking history
- Automatically ignored by git
- Backed up recommended

### Cache (`cache/`)
- Market data cache from `search.py`
- Can be regenerated by running `search.py`
- Automatically ignored by git

## Maintenance

```bash
# Clean old logs (older than 7 days)
find logs/ -name "*.log" -mtime +7 -delete

# View recent logs
tail -f logs/insider_scan_async.log

# Check state file size
du -sh state/*.json
```
