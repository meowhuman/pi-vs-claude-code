Spec written to `specs/daily-log.md` for the `extensions/daily-log.ts` Pi extension.

**Why:** User requested a structured work journaling extension with 4 tools, UX flow, and Markdown output format.

**How to apply:** When the Engineering Lead picks this up for implementation, refer them to `specs/daily-log.md`. The 4 tools are `log_entry`, `log_show`, `log_summary`, `log_search`. Log files live at `logs/YYYY-MM-DD.md`.
